# 商业可行性筛 (biz-feasibility)

把其作者沉淀的整套商业判断框架做成可复用评估器。**目的不是给点子打气,是用 kill-criteria 思维冷峻地证伪。** 默认怀疑,敢下"放弃"结论。

**认识论地位(强制 —— 报告开头必须自报家门,别冒充预测神谕)**:本工具是**决策卫生 / 红队协议**,不是经过样本外回测校准的预测器。它的价值在「逼出承重假设、构造反例、预承诺 kill criteria、把'小/封顶'从形容词钉成数字」,**不在**「我历史命中率 X%」。故:① 任何 KILL/PROMISING 都是**「基于当下可见结构与 fact 的裁决」**,不是「这事一定死/一定成」的预言;② 工具**最危险的失败模式是 false-kill**(把一个靠 niche / 创始人不公平优势 / 分发楔子能跑通的项目误杀)—— 末段红队必须显式问这一侧(见输出格式⑦「太软/太狠」双查 + 母级死因的 survivor-first 闸);③ **校准账(建议,非强制)**:下 KILL/CONDITIONAL 时记一行「裁决 + 日期 + 承重 swing + 复盘触发条件」,3–6 月后回看真实进展,积累 false-kill / false-promising 样本 —— 这是把「会解释已知失败」升级成「能校准未知成败」的唯一路径,也是本工具尚未补齐的短板,别假装它已补齐。

> 本框架由作者在真实评估中逐案打磨而成。文中 `A1–A14 / S1–S20 / X-CLEAN / DOG` 及 `代号 M/T/R/E`、`化名 J` 等都是**匿名案例代号**(原评估对象已去标识),只用于把抽象规则锚到具体「实犯」上 —— 删掉它们会让规则退回空泛、降低判准,故保留匿名版。公开的真实公司案例(LinkedIn / Clubhouse / Quibi / Jibo / Rover·Wag / Character.AI 等)均为公共事实,照引。

## 两种模式

- **快筛 (默认)**: 直接用下面 9 个筛子 + 死锁 + kill 阈值,口头给 verdict。几分钟。
- **深评 (用户说"深入/严格/多方案/帮我 brainstorm 并验证"时)**: 用 Workflow 跑 `feasibility-workflow.js` 模板(brainstorm 子代理出 ≥5 方案 × feasibility 子代理对抗式打分 × 母级死因逃逸裁决 × loop-until-dry),最后整合成完整报告。**只有用户明确要多智能体编排时才跑。**

## 第一性问题(先问这两个,定生死)

1. **「这门生意的供给,今天没有我也在开门吗?」**
   - 在 → **收割存量供给**,冷启动轻(只搬运需求)。
   - 不在,要我去点火 → **创造新供给**,冷启动是物理难题(要改变一群人的人生选择)。LinkedIn Career Advice 招专家死在这。
2. **「当我到达规模,会不会自动解锁一个能被我私有捕获的引擎?」**
   - 会(注意力→广告、graph→招聘、使用→数据,且锁在站内) → 公共品可被补贴成生意。
   - 不会(关系/价值会走掉、太弥散) → 永远是公共品,装不上引擎。Lunchclub/Clubhouse coffee-chat。

## 通用判断方法论(所有生意,不止平台 —— 别对非平台 freestyle)

⚠️ 真实 idea 流**大部分不是平台**(F&B/零售/服务/工具/分销/内容)。平台死因(去中介/新供给/引擎位置)对它们答非所问。**统一流程:先分类 → 跑通用骨架 → 挂该类型死因模块。**

**Step 0 分类**:平台/双边 · F&B/零售 · 服务/agency · 工具/SaaS · 分销/进口 · 内容/媒体 · **自营/自有资金投资**(自营交易、自买房收租、个人组合 —— **principal=customer,无外部付费方**)。

**先于分类的一问 —— 出钱方是谁(个人 vs 机构)?** 这条决定整套机会成本基准与四档语义。**默认是个人创始人**(本工具主场景);但若 idea 是**公司/机构的战略下注**(出钱方是公司、有团队和投资人的钱),则:①机会成本基准换成**企业资本 ROIC + 战略防御价值**(同样的钱和人投在别处的已验证回报 + 不做会不会被对手截胡核心),不是个人薪资;②四档做**企业版语义映射**——PROMISING=值得投/能防住或长出新引擎,副业·CAPPED=战略鸡肋(烧钱但回报封顶、防不住也长不大),KILL=该砍(既防不住核心又长不出独立引擎),CONDITIONAL=取决于某未验证 swing。报告里**点明用的是哪一版语义**。**大集团战略下注的死活准则**:活 = 满足「防御必要性(形态对症)/ 新引擎期权(期权尺寸 + fail-fast)/ 能力沉淀(产品死了组织仍得到买不来的资产)」之一 **且非僵尸**(无可证伪 launch 闸、靠高管赞助续命、长期不 launch、内部看衰 → 判死,哪怕养得起:Fire Phone / Meta metaverse)。判活用 **would-it-get-funded-as-a-startup**(同 idea 换独立团队融得到钱吗),不用「打得过成熟引擎近期 ROIC」(那样在位者永不投新东西)。

**先于分类的第二问 —— principal=人(技能/资金即产品)时,那个人的真实水平 / 本金 / track record 就是承重单位经济参数**(自营交易、单干咨询、个人投资:「这个人在不在那条尾巴里」= 整个裁决)。它载重 ∧ 高方差 ∧ 可查 → **必须问,不许用总体先验顶替**(原理见数据采集协议「先验不顶替可澄清的载重变量」)。
- **入场时间 / 排序结构本身是裁决信号(revealed preference)**:一个人**怎么进、何时进**暴露了他自己心里那个裁决。「拿 backtest 现在就 all-in 辞职」= 验证前下注 = **KILL 倾向信号**;「延后、对赌外部验证、保住主业、封顶本金」= 他**已在按副业·CAPPED 操作**,既是操作健康信号、又封顶裁决档位。读这个信号,别只读他嘴上的雄心。

**Step 1 通用骨架(6 节点,每个强制一个 fact/数字,不是形容词;缺则按 clarify-gate 问或标假设)**:
1. **需求拉 vs 推** — 点名今天**有人为它/替代品付的钱 + 价格**;「有货找市场」=供给推动(危险信号)。
2. **单位经济** — 套该类型公式(下表),算出**盈亏临界那个数**。
3. **频次/留存** — 客户**年化回来几次**。
4. **护城河** — 点名**最近的现存竞品 + 你结构性赢它的理由**(不是功能)。
5. **分发/冷启动** — **前 100 个客户从哪条渠道来 + 这渠道为何对你开放**。
6. **创始人适配** — 这门生意**瓶颈**要的那个资产,他有没有(逼证据)。

**Step 2 单位经济公式(按类型实例化)**:
| 类型 | 公式 | 致命比率 |
|---|---|---|
| 平台 | `Rev=base×转化×AOV×take%×频次×lifetime−CAC` | take / WTP |
| F&B/零售 | `月利=日客流×客单×毛利×天数−租金−人工−COGS` | 租金/营收比、翻台、盈亏客流/天 |
| 服务/agency | `=客户数×单值×毛利−交付工时×你的时间` | 劳动线性天花板(被工时锁死) |
| 工具/SaaS | `LTV=ARPU×留存月数;闸 LTV>CAC` | WTP(消费级≈0)、churn、被大厂吸收 |
| 分销/进口 | `=走量×单位毛利−物流−库存` | 需求拉动 vs 供给推动、商品薄毛利 |
| 内容/媒体 | `=DAU×留存×ARPDAU` | 留存引擎+分享环、为免费形态付费=WTP错配 |
| 自营/投资 | `净P&L=本金×净收益率(扣成本/滑点/资金成本/税后)`;闸=**真实样本外 edge** ∧ 容量 ∧ 机会成本回收比≥1 | 「验证态≠实盘」、过拟合、容量衰减(alpha 随本金涨而消失)、成本/滑点吃掉 edge |

**Step 3 死因模块库(挂骨架上,取代 freestyle)**:
- **平台** → 下面 9 筛子 + 三母死因 + 频次-密度-WTP 死锁。
- **F&B/零售** → 点位彩票、租金/营收比、人工密集、赛道饱和、无护城河、牌照。
- **服务/agency** → 劳动线性封顶、无网络效应、关键人依赖、项目 lumpy。
- **工具/SaaS** → WTP(消费级≈0)、套壳商品化、churn、单功能被平台吸收。
- **分销/进口** → 供给推动 vs 需求拉动、商品薄毛利、谁都能进货。
- **硬件/IoT**(机器人、可穿戴、智能硬件 —— 别折叠进「物理服务」,那条对硬件答非所问) → ①**BOM/零售毛利 + 退货率**(硬件毛利薄、退货吃利润);②**云依赖变砖**(产品靠云端续命,公司一死/服务器一关,已售出的硬件全变砖 —— Jibo 烧光 $73M 后用户手里的机器人集体变砖;问「谁永续供养这朵云」);③**购买驱动 vs feature 错配**(headline 功能未必是真购买理由 —— 宠物项圈车主为「丢狗恐惧」买 GPS,你引以为傲的「健康分析/行为模拟」被无视;先问「真金驱动购买的是哪个 feature」);④**公共品类硬件的补贴付费方依赖**(养老/健康/陪伴类硬件,使用方(老人)直接 WTP 弱 → 真付费方是 B2B/政府/保险,**付费方≠使用方**,套公共品补贴逻辑找补贴方,找不到=Jibo 化);⑤**医疗器械红线**(一旦带诊断/治疗/监测声明 → FDA/NMPA 器械监管,合规闸,可一票否决)。
- **物理服务** → **技能不可无人化**(应激/经验型劳动踢给用户=质量+责任双崩)、单位经济、合规。
- **内容/媒体** → 留存引擎 + 分享环;为免费内容形态做付费订阅=WTP错配(Quibi)。
- **自营/自有资金投资**(principal=customer,无外部客户) → ①**alpha 不存在/已被套利掉**(信号公开即衰减)、②**回测过拟合 / look-ahead / 幸存者偏差**(尝试越多 selection bias 越重)、③**无 live track record**(「数学验证 / 回测 ≠ 实盘挣钱」是承重墙,见口径锁定第⑦轴)、④**成本/滑点/容量**吃掉理论 edge、⑤系统性风险 / 杠杆爆仓、⑥数据与执行军备劣势(散户级数据 vs 机构 tick/alt)。**survivor(Renaissance 等)赢在系统性资产**(规模化数据+工程+资金+低成本通道+团队)**而非单人天才**——对照「用户一个人有没有这几条腿」,多半应回到机构容器内做,而非个人烧钱自营。**一旦收外部钱代客理财 → 触发牌照(CTA/RIA/MAS CMS),合规闸,可一票否决该路径。**

**体量自检(横切所有类型 —— 单位经济是体量的函数;体量 X 证明的结果/资产,不自动迁移到体量 Y 运营)**:任何 idea 判四件事 ——
1. **迁移方向(证据来自哪个体量?)**:benchmark / 回测 / track-record / 竞品 来自的体量 ≠ 用户实际运营体量 → 打折。**机构→个人**丢 stack / breadth / 组合分散 / 融资与佣金条件(实例 A14:公司回测与实盘是机构 stack + 组合体量、solo 是零售体量 → 不向下迁移);**大厂→小创业**丢品牌 / 分发 / 资本。**idea 能迁,economics 不迁。**
2. **下界(最低体量)**:某阈值下结构性不成立 —— 固定成本摊不平、**买不起 edge 所需的 breadth**(统计套利 IR ≈ IC × √breadth:本金小 → 头寸不够分散 → 特异方差盖过 alpha → 实现 Sharpe 远低于回测)、达不到最小头寸 / 保证金 / 准入门槛。**不是「赚得少」,是「那个数本身拿不到」。**
3. **上界(最高体量)**:capacity 衰减(alpha 随 AUM 涨而消失)、niche 饱和、创始人成瓶颈、监管随规模触发。
4. **放大工具自带风险**:杠杆放大尾部 / 爆仓(非线性 —— 无杠杆能扛的回撤,加杠杆变终局;crypto 叠加强平瀑布 + 资金费 + 对手方)、融资触发牌照、招人加协调成本 —— **用杠杆逃下界常只是放大「没分散的小账户」的方差,买不到 breadth,是陷阱不是解法。**
> 现有的 capacity-decay(自营)、频次-密度-WTP 死锁(平台)、劳动线性封顶(服务)都是本自检的**单格实例**;最易漏的是 **scale-DOWN 迁移损失 + breadth 下界**——「把机构的成绩单当个人的预期」。

**取证纪律(关键 —— 别读措辞,用 fact 下判)**:每个节点**能取证就取证**——深评里拉 macro 市场量、竞品坟场、行业 benchmark 比率,并**用网页/搜索核对口头 premise**。**核对的不只是数字,还有 idea 描述的对象本身**:若搜证发现「在做的事」与「brief 描述的事」矛盾,先把矛盾当头号 fact 暴露、对「描述版」与「真实版」分别判,别默默按描述版判完(按一个不存在的 idea 判得再精确也是废的)。**但要分清造假 vs 信息不对称**:若差异是「内部意图陈述」vs「隐身 / 预发布产品被刻意遮蔽的公开表面」(A13 ä»£å· M(æä¸å¸å¬å¸éèº«å©æ)),那是**信息不对称、不是造假**——掂量谁信息更多、标记不对称、回去澄清,别急着宣布「事实反转」。

**数据口径锁定(强制 —— 用任何用户给的数字前,先钉死口径,否则不许进单位经济)**:每个数字过七个 axis —— ①**时间单位**(周/月/年/单次)②**毛 vs 净**(营收/流水 vs 利润)③**单位 vs 总量**(单店/单团/单人 vs 全盘)④**累计 vs 活跃**(历史总数 vs 当前在用)⑤**一次性 vs recurring**⑥**撮合/动作次数 vs 付费人或单**⑦**验证态:回测/数学/原型/口头 vs 实盘/市场已付费**(「我跑通了/算明白了/做出来了」≠「市场为它掏过钱」——这条最常被偷换,且往往是整个 idea 的承重墙;凡 idea 重量压在一个只到「回测/原型」态的数上,标🔴未验证载重,先打掉它)。**口径不明且载重 → 必须问,绝不默认**;默认会按最乐观读 = 系统性高估或算错基准。实例:ä»£å· T(ææ®å app)「1000+」是撮合次数非付费会员;**A10 创始人「5000+」是「一周营收」,曾被误读成「一个月纯利」—— 时间单位错 ~4 倍 ∧ 把营收当利润,直接错算机会成本基准(⑥)**。快筛凭先验 + 标假设;深评取证。

**竞品探测纪律(白地是最该往死里搜的地方)**:创始人说"这里没人做"时,那块白地**恰恰要对抗式深搜**——杀手常是个你没听过的本土 niche 玩家蹲在缝里(实例:A1 运动场地,泛品类英文词只浮出 ActiveSG/Playo;搜创始人点名的"condo 约球"才挖到 **ä»£å· R(æ¬åçº¦ç app) 15.9万用户**,直接把 idea 判死)。三条:
1. **搜缺口、不搜品类** —— 直接搜创始人声称的那个具体白地,而不是泛泛的品类词。
2. **自适应 persona 搜索** —— 拿创始人给的目标用户,**假装成他、想象他的生活痛点场景,用他会 google 的话**搜多次(不是分析师的品类词);真实用户在用的竞品就会浮出来。
3. **多措辞 / 多语言 / 多角度**(by-用户痛点、by-使用场景、by-本地俚语),把竞品当**死亡坟场**挖,挖到连续无新增才算搜过。**只搜 1-2 次品类英文词 = 没搜。**
4. **结构模式 → 竞品集可推导,自主拉取(别等用户点名)**:brief 一旦描述出一个可识别的结构(**巨头生态防御 / 撮合平台 / 自营 alpha …**),其**同侪竞品集就是可推导的**——「还有谁在跑同一套剧本?」必须自己问、自己搜。**尤其战略 / 生态下注:首要竞品 = 正在跑同款的同侪在位者 + 他们【可观测的真实战果】**(实例:A13「巨头+AI助手+多业务生态防御」该当场拉阿里千问、字节豆包、腾讯元宝、Amazon Rufus —— 这些活体实验的结果就是最直接的证据)。
5. **分清「自主推理」vs「必须澄清的私有信息」**:竞品集、结构模式识别、母级死因 = 我**自主欠的推理**,做砸了就是推理失误;本金 / track record / 内部路线图 = 真正该向用户**澄清**的私有信息。**绝不用「需要澄清」给一次推理失误打掩护。**

## 9 个筛子(平台死因模块 —— 仅当 Step 0 分类为「平台/双边」时套用)

| # | 筛子 | KILL 信号 |
|---|---|---|
| 1 | **存量 vs 新供给** | 要逼供给方"开始营业" = 新供给陷阱(小市场尤其致命) |
| 2 | **频次/留存引擎** | 低频(年化 1-2 次)且无 习惯/网络锁定/积累价值 三引擎之一 → Clubhouse |
| 3 | **引擎位置(铁律1)** | 直接对 连接/展示/内容/学习 收费 → 必弱。引擎只能装在"才华可读→雇主/VC/交易"旁 |
| 4 | **去中介(铁律2)** | 收入挂在"一次性撮合"上(撮完加微信跳单 take→0) → 必死。须挂"持续高频行为" |
| 5 | **公共品补贴** | 答不出"谁补贴这个免费层" → 墓地;答不出"留存引擎" → Clubhouse |
| 6 | **平台化 vs agency** | 交付被人力线性约束、毛利不享网络效应 → 是 agency 不是平台 |
| 7 | **护城河 vs 强敌** | 差异是可复制功能而非结构壁垒(撞 GitHub/LinkedIn/Vercel/Upwork 等免费全球网络) → defensibility→0 |
| 8 | **wish-market-fit** | 需求暖而响 + 实付 WTP≈0(口头要、没人掏钱) → 别误判 PMF。供给侧也会 wish(供给空) |
| 9 | **数据引擎(若想卖数据)** | 不是"高频行为的被动尾气"而是"主动监控"(采集动作毒化行为/坦诚)、或转售公开数据无专有性 → KILL |

**营收公式**(给单位经济用): `Revenue = base × 付费转化 × AOV × take% × 频次 × lifetime − CAC`。WTP/take 是主变量。算 LTV:CAC,<1.5 = 警告,跳单会把 lifetime 砍到地板。

## 封顶筛子:频次-密度-WTP 死锁(才华/作品/人聚合类平台在小市场必撞)

> "才华可读→雇主/VC"这类**唯一合法的引擎位置本身是低频事件**(一个人几年才被雇/被投一次)。喂活低频引擎要靠**供给密度**堆规模;但小市场(如 SG)真实存量供给薄。于是两条路都通向死:
> - 要密度 → 自造/抓薄数据 → 买家 WTP→0;
> - 要 WTP → 收割真存量 → 市场太小撞 TAM 天花板(工作室不是平台)。
>
> **换遍所有供给来源,这堵墙不动 = 结构性死因,不可通过换角度消除。** 任何"作品/才华/人 聚合平台"在小市场,先用这条筛——过不了就别做。

## 平台/marketplace 专项:三大母级死因

2. **新供给陷阱**(逼供给开始营业;小市场=供给 wish)
3. **引擎装错位置**(对展示/连接/内容/学习直接收费)

## 数据分级与置信(强制 —— 所有结论必须数据导向,不许语义导向)

**铁律:裁决只能盖在 fact 上,不能盖在用户的措辞上。** 出任何结论前,先识别**商业模型核心载重参数**(那 5-8 个真正左右裁决的数字),对每个走下面流程。

**澄清门(顺序不能反)**:每个需要的数 → ①先判**是否载重**(用翻盘阈值测:这个数错了裁决会不会翻?)→ ②载重的**必须拿到**(问用户 / 网查),用户不给且查不到 → **标红**;非载重的才可**廉价假设**(橙)。

**数据采集协议(对 ideation 阶段用户降门槛,但绝不退回语义漂移)**:
- **前置闸(先问这一个)**:"你做过这类市场调研/用户访谈吗?有没有特殊供给/分发渠道或内部资源?" 两者皆无 → 默认**标配创始人 + 无内部渠道**,需求/转化类数字**默认用户不知道,我自己查/推**,不再追问一堆。
- **自查、不问用户**:可公开查的 macro/市场规模 → **我查**(用户永远不是市场级数字的来源)。**合规/法律/牌照 → 永远自己 cross-check,即使用户告诉我也不采信为 fact**(可一票否决,默认载重)。**查合规前先钉死法域**(创始人常驻地/资金所在/拟运营地)—— 不先定法域就查 = 张冠李戴(实犯 A14:误把创始人当在岸→套大陆 crypto 红线,实际在 SG/MAS,整条作废)。**侵权/安全/产品责任闸的强制触发器(不止 AI 陪聊)**:物理服务×弱势身体(老人/儿童/医疗·健康声明)、食品、活体动物、致死致伤可诉性 —— 命中任一**必须**跑这道闸,别只在 AI 案触发(实犯 S14 银发健身、S11 食安 漏触发)。
- **载重事实的方向性自检(强制 —— 防止把事实的符号接反)**:每个载重 fact(尤其政策/法规)落进裁决前,必须显式写出**它对需求/裁决的方向**,并做翻转测试 ——「**若此 fact 为真 vs 为假,裁决各往哪边走?**」两个方向都推一遍,确认符号没接反。**政策有两种角色,别混**:(a)**合规闸** —— 牌照/资质/红线 **+ 侵权/安全/产品责任暴露**(不止「有没有牌照」,还有「会不会被告死」 —— 未成年安全、医疗/健康声明、致死致伤可诉性;实例:AI 陪聊 app 无牌照问题,但 Character.AI 未成年自杀致死诉讼 + 意大利 Garante €500 万罚款,是侵权/责任面一票毒死 solo),缺则可一票否决(只会往 KILL 推);(b)**需求驱动 / 障碍** —— 一条政策可能是**利好**(移除一个抑制需求的顾虑 = 看涨),符号可正可负,**绝不默认「查到政策 = 约束 = 利空」**。**反例(A10,曾接反)**:创始人说「游学→某东南亚国家留学转化薄弱」,抑制因是家长怕「留学不能考编」;查证此说法**为假**(留服认证学历可考编)→ 正确方向是**障碍被移除 = 出海腿转化【可能被打开】= 看涨**,故 upside 分支**活的**,swing = 「把真相告诉家长后他们到底转不转」未测 → **CONDITIONAL(上摆分支存活)**;报告却把「抑制信念为假」错读成「卖点为假 → 判死」,符号接反。**凡载重 fact 是「障碍/顾虑/限制」类,先问:它若不成立,是利空还是利好?**
- **先验不顶替可澄清的载重变量(强制 —— 本轮反复踩的同一条)**:基准率 / pedigree / 身份标签都是**先验(待问的假设),不是对个体的测量**。用任何总体率前先**点名参照类、核对它匹配本案主体**(散户率 ≠ 职业 quant)。先验只**抬高提问紧迫度**,在**载重 ∧ 高方差 ∧ 可自查**的变量上**绝不替代澄清**(推断它 = 反 vibe 推断铁律所禁)。**两个推论**:① principal=人 时,人的真实水平/本金/track record 是承重参数,必须问;② swing 未测但可澄清 → 必须澄清或**保持 CONDITIONAL**,不准用坏先验塌成 KILL(除非无论 swing 怎样都结构性死)。**载重 fact 互相矛盾时,矛盾本身是要暴露的头号 flag、不是要抹平的噪声**(A14 频率:pedigree 偏 HFT vs 零售数据做不了 HFT → 必须问;问完「高频」还要 caliber-lock:微秒延迟套利 vs 秒级高换手 = 两个裁决)。
- **载重 = 高杠杆 ∧ 高方差 ∧ 不可自查**(三者全中才问用户)。高杠杆但**低方差**(烤串进价、同区租金,人人差不多)→ 用 benchmark,只问一句"这块你有没有特殊渠道?",无则标配,不深挖。
- **定性能定量就别要数字**(降门槛)——**但仅限**低杠杆/低方差参数,或 qual→quant 是**真查表**(城市名→人口);**绝不**凭语义 vibe 推一个**高杠杆高方差**的载重数(那就是退回语义漂移)。
- **未知太多 → 用上下界**:取认知范围高位/低位各算一遍裁决;**两界同向 → 结论稳健,不纠缠数据**;两界分叉 → 只追那个让它分叉的 swing 变量。
- **「未验证」≠「跳过算」——载重转化/WTP 标🔴时,仍必须锚一根 🟡 下界,别只算 founder 乐观上界就收**:可比品类的**真实付费转化** benchmark(转化率 benchmark 是低波动、🟢-可查的 fact,见下「翻案级闸」末句)× 法域折扣 = 1/X(就是「1/40×0.8=1/50」那一步),用它算**收入天花板下界 = SAM × 锚定转化 × ARPU**。**即便裁决是结构性死、转化不承重,这根下界锚也要给** —— 它把「封顶/体量」从定性骂街变成定量钉死(「乐观也就 ~N 付费人、年营收下界 ~SGD X」),比"WTP 未验证"一句空话狠得多,也堵住「只标🔴+只算上界」的偷懒。漏锚 = 放掉一个最便宜就能拿到的更狠下界(**实犯:X-CLEAN skill run 把转化留🔴/🟠、只算乐观上界 LTV:CAC、SOM 直接抄 founder「无转化锚」**)。

**用户叙述先分三层,别把后两层当 idea 证据(强制)**:用户给的每句话先归档 —— ①**fact**(可独立核对的事实);②**观察**(他对外部行为的可观测描述,如「6 个月多次掉头」,可当弱信号);③**自我感受**(「自认为方法新颖科学」「觉得方向对」——**从「idea 是否成立」里剥离,绝不当证据**;它本身常是 pre-live 过度自信信号)。但**自我感受的集群可反推 principal 的身份与阶段**(多条自信 + 资历语 → 「有真资历、backtest 跑通、深信却未被市场验证」的危险 pre-live 站位),喂回参照类与 founder-fit。

**4 档来源 + 载重标记(给核心参数上色,不是每个数字):**
| 色 | 档 | 含义 | 附录要求 |
|---|---|---|---|
| 🔵 | fact | 可独立核对的 macro / 可验证事实(如 SG 市场规模) | 标口径 |
| 🔷 | 用户断言 | 用户亲口说的,**载重则先廉价核对**(网页/反问),核过不矛盾才升🔵,否则降🟡 | 标"待核/已核" |
| 🟢 | referred | 网上查到,**须有源 + 源质量子级**(一线分析机构/primary > 口碑媒体 > 博客论坛) | 附录给**网址 + 原文引用** |
| 🟡 | inferred | 在🔵/🟢锚点上推导,**必须具名锚点 + 写明推导**(如 1/40 × SG折扣0.8 = 1/50);无锚点不许用 | 附录写为什么这样推 |
| 🟠 | assumed | 太线下/niche/用户不给,按宏观大概编 | 附录写**为什么找不到 + 为什么这么编** |

**🔴 = 派生报警态(不是第 5 种来源)**:`载重 ∧ 来源≤🟡`。触发则:①**报告第一句**标"结论可能因关键数据缺失不置信";②尽力去取(给用户**最便宜转蓝法**:"问家里人一个数 / 一次网搜就能转🔵");③实在无能为力则明说。

**翻案级 fact 复核闸(强制 —— provenance ≠ veracity,标了🟢+链接 ≠ 这事真存在且现行)**:源分级只查"出处",不查"真伪/现行";`🟢 + 像样 URL` 形式合规却可能是幻觉事实或被误读的真源(实犯:S7 编了"2026 和解"挂真格式假 URL;S13 把"Payoneer 对老卖家开收款"误推成"Etsy 重开中国新店",抹掉真死因)。故下列**高危易错类别**的 fact,**载重 / 当 headline / 当坟场 fact 用时,必须额外过真伪闸**:①**离散事件态**(和解/判决/收购/融资/关停/破产 + 日期)②**竞品死活态**(在世/已死/破产/被收/转型)③**现行可变数值**(关税/税率/价格/阈值/de-minimis 状态)④**法规/政策现行生效态 + 日期**⑤**平台/渠道现行准入规则**(开/关、允许/禁止)。复核动作三选一:
- **事件 / 死活态 → 原文回读**:必须能逐字引用源句 + 日期(≥2 独立源,或 1 primary:判决书/公告/财报);**引不出 = 你没有这条**。
- **现行可变数值 → as-of 标注**:标该值"截至日期"+ 注明可能已变;政策刚变 → 给区间、让**方向**而非精确数承重。
- **从单一源推出的翻案级结论 → 分两行写「源说 X / 我推 Y」**:Y 翻案级则 Y 本身需独立确认,不能借 X 的源混过(治"真源→错结论")。

**铁律**:任何**翻案级** web fact 过不了适用那项 → **降🟡 且裁决不得翻档压在其上**(转 CONDITIONAL / 标"待核")。**强制 final-pass**:出报告前扫一遍所有带🟢/🔵 的"事件/死活/现行值/准入规则",逐条自问"我能逐字回读源句+日期吗?这条现在还成立吗?"——引不出/不确定即按上降级(这一遍**不是可选**;实犯里 S15 自觉做了→躲过幻觉,S7/S13 漏了→失守)。与「载重 ∧ 高方差 ∧ **不可自查** → 必问用户」对称:此为「高杠杆 ∧ **高易错** ∧ **可复核** → 必复核 web」。**只对上述 5 类 ∧ 载重开火,低波动 fact(市场规模/转化率 benchmark)正常🟢通过,不过度开火。**

**post-cutoff / 近期事件 ≠ 幻觉(双向铁律,闸的两刃)**:训练知识里查不到 ≠ 它是假的。一条带**近期/未来日期**的翻案级 fact,**既不许凭记忆断言它真、也不许凭记忆断言它假** —— 只能 web 原文回读裁定(引得出真实源句+日期+≥2 源 → 采信并标 as-of;引不出 → 降🟡,而非宣布"编的")。**「我没听过/我的知识里没有」对一个 post-cutoff 事件是零信息,不是证伪信号。** 实证:S7「2026-01 Google/Character.AI 和解」一审被(连同评审者)误判为幻觉、其实 web 可证为真;同一次闸真正拦下的是对**真源的夸大 framing**(「法官裁定 chatbot=product 非 speech」实为「仅就产品责任目的、且法官明说此阶段不定论」)与引不出原文的 Garante 罚款(降🟡)。**此铁律对评审/复核自身同样适用** —— 说"这条是 skill 编的"本身就是个翻案级断言,不 web 回读就下 = 评审在犯它要抓的同一种错。

**「先行者/已做成」断言 = 载重,必须核它今天的死活(死活闸的适用对象扩到 founder 引用的 precedent,不止直接竞品)**:当 founder 用「**X 已经做成了 / 海外先行者验证过这个模式**」当 reason-to-believe(本案「美国 Rover/Wag 做成了」),**那家先行者今天还活不活本身就是载重 fact** —— 它是 founder 信念的承重柱,塌了就直接反证前提。故必须把上面「②竞品死活态」的核查**主动指向这个 precedent**(不能像核直接竞品那样顺手核了、却把 founder 的「先行者」当背景常识放过)。两条动作:① **用当下状态 query 核**:`实体名 + 当前年份 + status / bankruptcy / shutdown / layoffs / acquired / 破产 / 关停`;**禁止拿 evergreen 对比页(`X vs Y cost`、`rover-vs-wag`、融资历史页)当死活来源** —— 那类常年不更新,会把已死实体显示成 survivor(正是失守口)。② 先行者**已死 / 已萎缩 / 已被贱卖** = 把它从「模式被验证」翻成「模式连先行者都没撑住」,是反证 founder 的硬料,必须写进裁决。**别指望跑它的模型"碰巧记得"** —— 有 recency prior 的模型(本案 codex)会主动去confirm,没 prior 的不查就漏;闸的作用就是无论模型记不记得都强制查。**实犯:DOG-skill run 引 Wag ~$300M 融资、把它当 survivor 列进「美国寡头」,漏了 Wag 2025-07 已 Chapter 11 破产(用了 evergreen 的 NerdWallet rover-vs-wag 页);codex 发当下状态 query 抓到了 SFGate 破产报道。**

**适用性闸(强制 —— provenance ≠ veracity ≠ applicability,三道不同的轴)**:一个数标了🟢、也核了真伪,**仍可能套错对象**。承重数落地前再过四项匹配:①**法域匹配**(美规 APHIS/ABA/UPL 不许套中国/英国案;**合规闸开火前先钉死创始人常驻地/资金所在/拟运营地,再查那个法域** —— 实犯 S19 APHIS、S20 ABA、A14 误设在岸);②**子类匹配**(类别均值 ≠ 子类:食阁泛摊 10–15% 毛利不许直接套烤串 —— 实犯 A6;要么按子类成本线重算,要么标"未验证迁移");③**口径匹配**(毛 ≠ 净、累计 ≠ 活跃、单店 ≠ 全盘:>70% 必标"毛利非净"并附净额);④**在位者地理匹配**(把"在位者"落到创始人**买方地理**的真实玩家:杭州→美团/美管加而非 Fresha;并查"在位者是否已免费打包你的差异化" —— 实犯 S16/S8)。不匹配 → 不许直接套用,降🟡 或按子类重算。

**报告置信度 = 载重参数里最弱的那一档(取 min,木桶,不是计数)。** 任一载重项🔴/🟠 → 整篇标低置信。非载重橙色占比 > 一半 → 标"仅定性方向"(次要卫生指标)。

**裁决继承输入置信度**:KILL/PASS 压在🔴上 → 只能叫"暂定 KILL,待 X",别说得比输入更确定。

**商业模型 section(每份报告必有)**:写清这门生意的模型 + 核心参数,**每个参数标上面的色**,让用户一眼看出哪根支柱是 fact、哪根是编的。有多步漏斗 / 翻盘故事时上**可视化**(漏斗瀑布 / 敏感度龙卷风 / 单位经济桥)。 **承重 tag**:商业模型 section 把哪些参数「承重」显式 tag —— 凡出现在 BLUF / verdict vector / 死因段这些 load-bearing 面的外部 fact,没 tag 的按未 tag 承重处理、弱源则不许压裁决(防「弱源不标承重、叙事仍靠它钉结论」)。

## 判前 input-audit 闸(强制顺序 —— 下判前逐条过,别等用户事后澄清才补)

**多数误判不是丢了信息,是当场判错了面前的信息**(参照类选错、把自我感受当 fact、政策符号接反 —— A10/A13/A14 全是这类)。这些 skill 各处都有规则,但靠用户事后抓 = 太晚。故下任何裁决前,对每条**载重** utterance 强制走完五问,缺一不许进单位经济:

0. **输入隔离闸(最前,强制,硬 BLOCK 级 —— 治串案污染 / 脱敏复原 / 跨 case 渗漏)**:下判前先钉死「本 brief 实际给了什么」。报告里出现的**每一个具名实体(人 / 城市 / 公司 / 行业 / 项目 / 过往方向 / 机会成本基准)**必须满足三选一:**① 本 brief 明确给了;② 我 web 查到并附了源;③ 标了「我的推断」**。三者都不是 → **外部渗漏(别的 case / 别处会话上下文 / 模型联想),删除**。两条硬子规则:**(a) 脱敏不可复原** —— brief 把城市 / 亲属 / 公司脱敏了(如"某中部三线城市""我家人"),报告**不许猜回 / 填回**具体值,更不许拿脱敏前的真名做 macro 查证(实犯:A10 brief 已脱敏成"某中部三线城市""我家人",报告却把它们复原成具体城市 / 亲属并据此查 macro);**(b) 跨 case 不串味** —— 本工具在别的评估 case / 别处会话里见过的方向 / 资产 / 历史结论,**绝不进当前这份 founder 报告**(除非本 brief 自己给了)。命中任一 = **硬 BLOCK**(非 partial),即便不改主裁决也不放行。**本闸静默执行(强制)**:它只在内部决定"某实体删不删",报告**绝不**书面提及本闸 / 任何被排除的方向 —— 连否定式("我不引入 X""不串案""那不是你的方向""别处那个已验证方向")都算点名泄漏。报告只**正向**陈述本案实际用的基准与实体,不留任何"我排除了什么"的元叙述。
1. **三层归类** —— fact / 观察 / 自我感受?(自我感受绝不进证据)
2. **参照类点名 + 匹配本案主体** —— 用任何基础率/pedigree 前先点名参照类、核对它匹配本案主体(散户率 ≠ 职业 quant、泛摊 ≠ 烤串、机构回测 ≠ 个人实盘)。
3. **符号翻转测试**(该 fact 若是障碍/政策/顾虑/限制类)—— 「它为真 vs 为假,裁决各往哪边走?」两向都推,确认没把利好接成利空。
4. **验证态 caliber-lock**(口径第⑦轴)—— 回测/数学/原型/口头 ≠ 实盘/市场已付费?「描述的对象」=「真实在做的对象」(造假 vs 信息不对称)?
5. **载重 ∧ 高方差 ∧ 不可自查 → 必须澄清,不许用先验顶替**;载重低方差 → benchmark;非载重 → 廉价假设标橙。

**分工**:此闸治**遗漏型**错(治"第一遍少错");末段红队(⑦)治**信念型**错(治自评盲区);跨案审计治**漂移**。三者别混。此闸不加新知识,只把散落的规则**钉成下判前的强制顺序**。

## 触发型专项闸(按 brief 命中才跑,别每案硬套)

- **渠道闸(channel-fit)** —— 触发:brief 指定首发/主渠道(iMessage/WhatsApp/小区群/Word·CLM/微信/App Store…)且该渠道影响分发/合规/形态。单独成段判:它是**分发渠道、只是交互通道、还是增长飞轮**?**允不允许**这个产品形态(审核/身份/商业账号/年龄/隐私/人工转接)?用户**为什么**在这渠道接受陌生产品?**只是交互通道就不得计入分发优势**(实犯:A13 把 iMessage 折进整体死因没单独判;Apple Messages for Business 是审核制商业通道,非 Gen-Z 社交增长飞轮)。
- **战略迁移闸(strategic-migration)** —— 触发:叙事是「先 A 市场/用户/场景跑通,再迁回 B 核心生态」(尤其企业战略下注)。必须证 **A↔B 可迁移**:A 用户与 B 核心用户重叠?A 数据能迁?A 留存能转成 B 战略防御?A 能力被 B 复用?**证不出 → A 成功只算普通 consumer 成功,非战略成功**(实犯:A13 美国陪伴 bot 留存好 ≠ 能防东南亚电商/支付/游戏入口)。
- **合规措辞精度闸(legal-wording)** —— 触发:牌照/法律/金融/医疗/未成年/跨境/教育结果。**不许「无牌照门槛」写成「无责任/无信任门槛」**(实犯:S20「unreserved → 律师资历被证伪」;应「非法律准入护城河,但仍可能是信任/销售/质量/责任资产」);**不许「可合作」写成「挂靠」**(实犯:A10「挂靠旅行社」与自家附录自相矛盾;应「持证主体作合同/收款/出团/保险/责任主体,founder 仅获客或授权协助」);**不许「纠正误解」写成「销售承诺」**(实犯:A10 考编;应「只纠误解、不承诺结果、以当年招录公告/专业目录/当地政策为准」)。分清 legal-permission/trust/liability/insurance/procurement/professional-responsibility。
- **赛道死活措辞闸(category-not-dead)** —— 触发:某赛道既有死亡案例又有活跃收入/增长玩家。**不许写「赛道在死」**,除非同时命中多条(需求萎缩 ∧ 监管压死 ∧ 头部撤退 ∧ 客户预算消失 ∧ 分发关闭 ∧ 多家 survivor 同期转型/裁员/破产)。否则写「**赛道有需求,但本案缺该赛道可行玩家的逃逸资产**」(实犯:S20「AI companion 赛道在死」过绝对)。

## 输出格式(verdict)

给结论时包含: ①BLUF 一句话裁决(**PROMISING / 副业·CAPPED / KILL / CONDITIONAL** 四档,敢说 KILL;不再用 WEAK)+ **裁决对象钉死 + verdict vector(强制于 KILL / 副业·CAPPED —— 防「单标签覆盖全域」的神谕读法)**:主裁决之外,BLUF 内或紧随其后必须列一张 verdict vector,把主裁决**绑到它真正适用的对象上**,并对相邻子问分别给档 —— 例:`当前对象(某具体形态)=KILL;创始人全职=KILL;venture-scale=KILL;平台化=KILL;最便宜 wedge(concierge/微生意雏形)=CONDITIONAL→封顶至 CAPPED;boutique 微生意=CAPPED(对本 founder 不值全职);community/声誉期权=超出本裁决范围/未裁`。**这不违反「一份报告一个裁决」**——那条禁的是两个**相互矛盾**的 BLUF;主裁决仍唯一,vector 只是把它的**适用域钉死**,防止「KILL」被读成「所有形态都死」(把 §⑧ 可逆图与两轴解耦提到 BLUF 前台,而非藏在文末)。 ②命中了哪些筛子/死因 ③单位经济示意(营收公式+LTV:CAC) ④what-would-have-to-be-true(要成立必须为真的 2-4 个关键假设) ⑤**最便宜的证伪测试 + kill criteria**(优先 concierge/零开发,先打掉最致命的未验证假设) ⑥**与用户当前最优替代的相对吸引力(机会成本基准)** —— 基准是**用户自己的**:现有主业 / 已验证的其他方向 / 全职薪资,不预设任何具体方向。问的是"这门生意值不值得从那个基准切过来";若只清得掉现金成本、清不掉这条机会成本线 → 就是「副业·CAPPED」而非 PROMISING。用户没给基准就用全职薪资当默认锚。**串案污染子闸(内部静默)**:基准**必须取自本 brief 内 founder 自陈的方向**;没给 → 默认全职薪资。**绝不把 brief 之外的方向当基准** —— 尤其本工具**在别的评估 case / 别处会话里见过的方向**(无论它具体是什么):那是别的语境的方向,拿来当基准 / 比较 = 串案污染。**此自检只在内部跑,绝不写进报告**:报告只正向写「本案基准 = {founder 自陈 / 全职薪资默认锚}」,**禁止**出现"串案污染子闸""别的 case 那个方向""不引入 X 方向""不串案"这类元叙述或否定式点名(那本身就是泄漏)。引不出本 brief 的基准就写"brief 未给机会成本基准,用全职薪资默认锚",到此为止。**若出钱方是公司/机构(战略下注),基准换成企业资本 ROIC + 战略防御价值,四档走企业版语义**(见 Step 0「出钱方是谁」一问)。 ⑦ **第三方一眼 + 红队收尾(强制末段,形状必须可变 —— 固定三段式会被解码成模板)**:每份裁决收在一段自我挑战 ——(a)**第三方一眼**:一个没读过我推理链的局外人会说"显著该冲"还是"显著会死"?我的裁决与这一眼差在哪、凭什么?(b)**双向红队**:不只查"太狠了吗"(把可澄清 swing 塌成 KILL),**更要查"太软/太对冲了吗"**(modal 已重度加权却写成开放 CONDITIONAL、或现状=KILL 却写成 CONDITIONAL 化妆)—— **历次收口只单向纠过"太狠",此段必须主动补"太软"那一侧**。**且断言「无 false-kill」必须分级 + 限定对象(强制)**:禁止裸写「没有 false-kill」。只能写「**在【裁决对象】上无 PROMISING 级 false-kill**」,并显式承认是否仍存在 **CAPPED 级松动面** —— 未测 swing 救不了 venture/全职,但可能把最便宜 wedge 从 KILL 抬到 CAPPED;那**不是「裁决错」**,是 verdict vector 里 wedge 子档本就为 CONDITIONAL→CAPPED。写明它,别用「无 false-kill」一句盖掉。(c)**塌缩测试**:哪个单一 fact 一翻就换档?真无可挑就明说"干净裁决,无可挑"——**这个非触发正是它不是模板的证明**。若本案触发了独立 red-team,其结论并入此段。

**交付可读性(强制 —— 报告是给人看的,不是给框架看的)**:
- **① 说人话**:正文每个硬结论(尤其封顶/KILL)先用大白话给出。**内部框架术语(breadth / Grinold / 轴1轴2 / modal / revealed preference / wish-market 等)不许在正文裸用** —— 要么换成大白话,要么首次出现时一句话解释,公式/推导细节进附录或单独「怎么算出来的」小节。
- **② 展示推理,不只甩结论**:任何「封顶 / 清不掉 / 结构性死」这类断言,后面**必须跟一条可读的 why 链**(三步内、讲人话、看得见从 fact 到结论怎么走)。**只给结论 + 一个公式 = 被读成断言,内行不信**(实证:A14 报告把 breadth 数学压成一行术语,内行 J 读不出「讲道理」在哪、当场不信。对照:codex 白话 + 标假设,反而显得可信)。**对内行用户尤其:结论可以硬,但推理必须摊开、可独立检验。**
- **③ 激进结论必须挂在承重 fact 上**:任何 KILL 味 / 一票否决的结论,**显式写出「它压在哪一条承重 fact 上 + 那条若变成 X 则此结论作废」**,让读者一眼看出结论的依赖与脆弱处,而不是感到「不问就硬断」(实证:A14 把「撞大陆监管红线」硬断,没挂在「创始人在岸 vs 离岸」这条上;创始人实际在 SG → 整条本应作废,却被读成全篇激进)。
- **④ 用用户自己的语言出报告(强制 —— 本工具面向多母语市场)**:报告正文**默认用用户输入/澄清时所用的自然语言**(中文 pitch → 中文报告;English pitch → English report;西/日/印尼语同理),镜像用户的语言而非评估者的默认语言。**判定**:看用户**主体叙述**用哪种语言(夹带的英文专有名词/产品名/行话不算切换信号);用户显式要求某语言或要求双语 → 听用户的。**四档裁决标签(PROMISING / 副业·CAPPED / KILL / CONDITIONAL)与框架内部术语属技术锚点,可保留原文,但首次出现必须用用户的语言一句话解释**(同①)。**fact 墙的来源链接/原文引文照原样保留**(provenance 不翻译,免得引入误差),但每条的中文/英文解读句跟随报告语言。目的:开源后东南亚/多语种创始人拿到的是母语可读报告,不是隔一层翻译的二手判断。**别用「我习惯中文」当借口默认中文**——和「先验不顶替可澄清的载重变量」同源:评估者的默认不能盖过用户的实际语言。

**核心姿态**: 不 all-in 任何单一方案;先用最便宜的实验打掉唯一致命的未验证假设,再决定资源分配。需求越暖越要警惕。

**裁决分四档,每档映射唯一动作(WEAK 已拆 —— 一个标签映射两个相反动作本身就是缺陷)**:
旧 WEAK 含两义「封顶」与「在失败」,二者是**完全相反的决策**(封顶=做、当副业;在失败=别做、会失血),同名即误导。拆开后会发现「在失败」根本不是独立档——**它就是 KILL,只是死得慢**;真正需要单独成档的只有「封顶」。故收敛为四档:

**两轴解耦(强制 —— 别把「该不该做」和「能不能放大」绑死)**:四档其实压着两条独立的轴 ——(轴1)**这门生意清不清得掉【本 founder 自己】的机会成本** = 对他个人的 go/no-go;(轴2)**可不可投 / 可不可放大** = 是不是 VC 级 PROMISING。**机会成本基准是该 founder 自己的真实最优替代,不是统一全职薪资**:大厂裸辞的人基准是大厂包;职校毕业首店、返乡青年、宝妈、应届生的基准低得多。**一家清得掉 24 岁技师机会成本、但不可放大的单店,对她就是「该冲的主业」,却仍是 CAPPED(不可投/不可放大)** —— 这不矛盾,是两轴各自的取值。**「副业·CAPPED」里的「副业」二字只描述轴2(封顶、不可放大),绝不读成「次等/兼职/对谁都是 side gig」**;对低机会成本 founder,它可以是「做,这是你最优选」。下判前先把这个 founder 的机会成本锚显式设定(它是最 swing、最因人而异的输入)。

| 裁决 | 含义 | 唯一动作 |
|---|---|---|
| **PROMISING** | 清得掉机会成本,可投 / 可放大 | 上,可 all-in / 融资 / 连锁 |
| **副业 / CAPPED(微生意)**(= 旧 WEAK 的「封顶」义) | 清得掉现金成本 ∧ 不可放大/不可投;**是否清得掉机会成本要看【本 founder】的基准** —— 对大厂裸辞者清不掉(故只配当副业),对低基准 founder(职校生/返乡/宝妈/应届)可能清得掉、就是他该做的主业 | **绝不 all-in / 融资 / 连锁**(因不可放大);但**做不做取决于本 founder 的机会成本**:基准高→只当副业,基准低且清得掉→可当主业做。标签的「副业」只指封顶,不指次等 |
| **KILL**(吸收旧 WEAK 的「在失败」义) | 连现金成本都清不掉(慢性失血)/ 结构性不成立 | 别做 / 退出。尸检点明是「结构性死」还是「慢性失血」 |
| **CONDITIONAL** | swing 变量未测,决策树路由到上面三档 | 去测 swing 变量;分支结局 = PROMISING / 副业 / KILL |

> **「在失败」≠ WEAK 的一种,它就是 KILL**(死得慢而已);只有「封顶」单独成档,叫「**副业 / CAPPED**」。CONDITIONAL 的「副业」分支一旦被 swing 变量坐实,就落地成「副业 / CAPPED」档——三分支精确等于三个顶级裁决,不再有「副业是某档子义」的绕。**报告里写清它过的是哪条及格线**(现金 / 机会成本 / 可投放大),别再用一个模糊标签把"当副业"误读成"会死"或反之。「WEAK」一词已弃用。

**一份报告只有一个裁决(强制)**:复核 / 补取证若改变裁决,**必须回改正文 BLUF + 标题**,不是只在文末追加一段新裁决 —— 头尾两个裁决并存 = 报告自相矛盾、自毁可信度。附录可增量追加,**裁决不行**。

**CONDITIONAL 裁决格式(强制 —— 硬化后 CONDITIONAL 是高频输出,它就是「给用户一个精确的 next step / checking action」,必须写成一份可执行的验证规格,不是一句「去验证」)**。六件,缺一不算合格,但**第 0 件先判**:

0. **先声明 modal 权重(最前,强制 —— 防 CONDITIONAL 注水成"不想下结论"的安全档)**:CONDITIONAL 分两种,当场分清并声明 ——(a)**三档大致等概率的真开放摇摆**(A8/A12 型:投资前、swing 未测、廉价可测)→ 正常写决策树;(b)**已有一个重度加权的 modal、只剩一条小 upside/downside 支**(A10/A14 型:modal 其实是副业·CAPPED;A13 型:现状其实是 KILL)→ **先报那个 modal 档当主裁决,再把 CONDITIONAL 降级成"那条支的附加测试"**,不许用 CONDITIONAL 把 80/20 的局伪装成 50/50。**判据**:若 revealed preference / 结构约束已把某一档压到明显最可能(他已在按副业操作、或现状三条僵尸信号全中)→ 那一档就是主裁决,CONDITIONAL 退居其后。
1. **第一句 = 决策树**:「你现在必须验证 {swing}。若 X → PROMISING;若 Y → 副业·CAPPED;若 Z → KILL。」swing = 上下界**分叉处**唯一让裁决摇摆的载重数,摆最前,别让用户读完全文才知道验什么。
2. **swing 凭什么是它(一句推理)**:点明**上界假设 vs 下界假设各是什么、为什么裁决正好在这个数上分叉**——让用户看到它是上下界唯一的岔点,不是随手挑的。
3. **分支阈值必须是数字,且每个从单位经济推导**:不许「转化好/不好」。要「X ≥ T₁ → PROMISING;T₂ ≤ X < T₁ → 副业;X < T₂ → KILL」,并注明 **T₁ = 清掉机会成本(⑥)所需的那个数、T₂ = 盈亏临界 / 清现金成本(③)的那个数**——阈值锚回单位经济与机会成本基准,别凭感觉给。
4. **最便宜的取数动作(operationalize —— checking action 的肉,接 ⑤)**:具体到 **测什么指标 · 要几个样本 N · 用什么零开发手段(concierge / 预售 / 小额实盘 / 打 N 个电话)· 多久 D · 花多少 $C**;优先一周内零开发能拿到的。**对「验证态」类 swing(自营 OOS edge、留存、转化)再加三条**:(a)**pass 条件锚在独立样本数 N≥k,不是日历时长** —— 时长 D 是**频次的函数,先澄清频次**(实证 A14:周级策略 12 周根本攒不够 N,「12 周」不是通过线);(b)**测试本金/规模给下界公式,不编整数**(从单位经济/breadth 推:本金太小→头寸不够→噪声盖过 edge),数字缺创始人参数就标🔴待填,**绝不瞎填一个 plausible-precise 的整数**;(c)**禁止把「过线后的加码阶梯」写进测试** —— 分阶段上规模是**部署/人生决策(M4 侧)**,不是证伪测试;测试只到「用最小有效规模看清 edge 在不在」,加码留给用户。
5. **结果 → 动作决策表(预承诺)**:一张小表「实测落哪个区间 → 立刻做什么(升档 / 封顶 / 退出)」,事先钉死、不可事后谈判。
6. **边界守卫(M4,不可越)**:动作只到「测这个 idea 的 swing 那个 fact」,**不到「你该不该把人生 / 主业押上去」**;把岔点、阈值、最便宜实验钉死 = 把刀磨利,但**「要不要从机会成本基准切过去」的决定留给用户**——不替他选载体、不替他下人生判断。

**kill-criteria 设计纪律(强制 —— 别把 N=10 的硬阈值当统计神谕,也别把 metric 选错)**:concierge/零开发测试是**廉价方向性证伪**,不是有统计功效的实验。故:
1. **测机制,别测单一硬阈值**:把「跑通 vs 没跑通」拆成可读机制信号 —— **供给复用率**(同一供给愿不愿接第 2、第 3 个陌生 brief)/**需求复用率**(同一用户下次有新问题还回不回来找你)/**支付位置**(用户愿在 intro 前付、session 后付、还是只肯 free)/**跳单后是否仍有新 match 需求**/**每成功一单创始人花多少分钟**(人工时耗)/**愿不愿推荐给同事**(匹配质量)。每条给方向,不止给一个 pass/fail 数。
2. **暖样本不对称(关键 —— N 小但仍可下 KILL 的理由)**:冷启动样本多来自创始人自己人脉 = **友好样本**。友好样本里**失败 = 强 KILL 信号**(连最容易的都过不了);但友好样本里**成功 ≠ 可升档**(可能只是朋友捧场)。故阈值天生**不对称**:低分→KILL 合法、即便 N=10;高分→只松动到 CAPPED,**绝不**因为「友好样本过了」就升 PROMISING。报告里写明这条不对称,别让读者把「N=10 过了」读成「市场验证了」。
3. **去中介别用「裸跳单率」当 metric**:跳单 ≠ take→0,**前提是每个新需求都是新问题、新对的人**。真正承重的是**需求复用**——「用户下次有**不同**的新问题,会不会还回来让平台找**另一个**人」。若收费发生在每次 intro 前,单次跳单只是复购损失、不是核心漏。把 metric 从「跳单率」改成「需求复用率 + 收费位置」。
4. **N 小就明说功效有限,别伪装精确**:写「N=10 是方向性信号、不是统计显著」,给的是「往哪个方向偏」而非「精确通过线」。
5. **测逃逸,不只测死因(强制 —— 治「实验只复现我已信的死因、没测我自己承认的唯一逃逸路径」)**:每个 KILL/CAPPED 的最小测试分两支 —— **death-test**(复现死因)+ **escape-test**(去打 survivor-first 闸/可逆图点名的那条唯一逃逸路径,看能不能逃)。只设 death-test = 确认偏误的实验版。**pass 阈值必须 decision-changing**:禁「非零信号/有人感兴趣/愿意试用」当通过线(几乎必然出现、改不了任何决策);阈值锚到「过了就升档、不过就维持/降档」那个数。**escape 过 → 至多升到 passed-state ceiling(写明哪档),不自动升 PROMISING;哪些维度 escape 过了仍 KILL(non-upgradable)也点名**。(实犯:S9 只测跳单/需求复用,没测它自己承认的"managed workflow retention";A13"非零电商意图"太弱;S20 没测 B2B 采购/安全/责任闸。)

**CONDITIONAL 不可被先验塌缩**:见数据采集协议「先验不顶替可澄清的载重变量」推论② —— swing 可澄清(问得到 / 查得到 / 即将被别人验证)就别用坏基准率塌成 KILL,只配一个**怀疑型决策树**;除非结构性死(那要写明「与 swing 无关」)。

## 附录:竞品 / 市场 / 别人的来时路 —— fact 墙(每份报告必有,强制)

⚠️ **正文给裁决,附录给 fact 弹药。** 用户最缺的不是结论,是**「别人是怎么走的、活的怎么死的、市场到底多大」这些可independently 核对的硬料**。每份报告**末尾必须单独放一块纯 fact 墙**,只放可引用的事实,**不放任何我的推断/裁决/形容词**(推断留在正文,这里只有 fact + 链接)。

**硬性格式**:分门别类,**每条一个 bullet = 一句话事实 + 一个 reference link**。链接必须是真实可点的 source(一线机构/primary > 媒体 > 论坛);查不到链接的条目**不许进这块墙**(那是推断,归正文标🟡/🟠)。每条句末标来源色(🔵/🟢)。

**分类骨架(按 idea 取用,空类删掉,别硬凑):**
- **市场规模与趋势** —— TAM/SAM、增速或缩水、客单价 benchmark(各一句 + 链接)。
- **活着的竞品** —— 每个直接/间接竞品:是谁、规模/用户数/营收若公开、它靠什么活(商业模式一句话)+ 链接。**白地深搜挖到的本土 niche 玩家必须列**(如 ä»£å· R(æ¬åçº¦ç app)/ä»£å· E 这类正文点过名的)。
- **死掉的竞品(坟场)** —— 谁做过同类、死于什么、死时规模(各一句 + 复盘/报道链接)。**这是用户最该看、平时最难自己挖的料,优先写厚。**
- **别人的存活/教训(来时路)** —— **不止「谁活着」,要给 survivor 真实的单位经济 playbook**:它靠哪条腿 / 哪个资产把单位经济做正(各一句 + 访谈/帖子/财报链接),并**把 survivor 的制胜资产对照「用户有没有」**(接回 founder-fit 节点)。这是 fact 墙里最该深挖、最服务 go/no-go 的一类。
- **监管 / 牌照 / 合规** —— 与本 idea 相关的硬规则(各一句 + 官方法规链接)。**这块永远自查、必给官方源,不采信用户口述。**

**取料纪律**:这块墙的厚度直接等于报告价值——**深评必须真去 web 搜竞品坟场和市场数,不许凭先验填**。挖到连续无新增才算搜过(同竞品探测纪律)。宁可一类只有 2 条带硬链接的 fact,不要 10 条没源的断言。**每条标源质量子级**(一线机构/primary > 媒体 > 论坛博客;论坛/第一手运营者口径要标明 —— 常比设备商/卖方研报更可信,别因为是论坛就低估)。**fact 墙挖到的载重 fact(尤其合规/政策)必须回流进正文承重墙与裁决,不能只躺在附录 —— 附录不是裁决的垃圾桶。**

## 责任边界(M4 — 只判,不导)

工具的职责**止于裁决 + 关于用户「行为 pattern」的一条结构性观察,不越一寸去开人生/职业处方**。

- **IN-SCOPE**: 冷判每个 idea;指出可救内核;连环全 KILL 时,至多丢**一句可证伪、可引用、留在商业判断能力内**的观察 —— "你已送 N 个 idea 来被杀,pattern 本身就是数据;没有一个是'那件事',想创业这个动作才是。" 这是「核心一问」**唯一可防御的形态:一条关于被观察行为的陈述,用户自己完成领悟**,绝不替他诊断内心、绝不开载体。
- **HARD BOUNDARY**: 一旦问题变成"那我该做什么 / 要不要创业 / 我是不是完了" → 声明能力边界("我是商业可行性筛,对你的人生方向没有方法,硬答就是 guru 汤")并 refer 给真人。
- **绝不可跨**: 任何关于用户**内心**的断言("你其实是想解决焦虑")、任何**推荐载体**("转岗/公开写作/周末项目")—— 二者按本工具自己的本体论都是不可证伪的 wish-market claim,说出来就向后毒死每一个真裁决。
- **Duty-of-care(两条,属 in-scope,因为是澄清裁决范围而非延伸它)**: ①一句"被 KILL 的是你的 **idea**,不是**你**";②**危机语言 tripwire** —— 出现"没救了/不想活"等绝望词,立即停掉通用 referral,升级到真人 handoff。

## 交付纪律:正向反馈怎么给才不毒化 KILL

裸冷水会触发 wish-market 用户的**心理逆反**(防御、否认、离开)。但解法**不是软化裁决**。统一铁律:

> **正向反馈能扛住防御型用户,当且仅当它是关于「交易」或「idea 这个对象」的可证伪断言,绝不能是关于「人」的断言。这和杀 idea 用的是同一条边界。**

**三个必过的存活测试**(缺一即退化成参与奖、反被识破):
1. **可证伪** — 用户拿自己历史一对就成立(夸"你在变强"会被他的 KILL 计数当场证伪 → 禁用)。
2. **In-scope / 非人身** — judge 交易/动脉/组件,**不下身份判决、不跨次排序、不庆祝**。
3. **裁决中性** — 让用户**花更多不是更少**(更利的刀=严谨;免费安慰=推销)。
> 目标不是"躲过识破",是"**经得起被识破**":真、in-scope、不奉承的话,意图被看穿也无把柄。

**KILL 输出四拍**(无暖场过渡;**形状必须可变,任何"每次都触发"的结构会被解码成脚本,反噬裁决诚实**):
1. **裁决** — 裸冷 KILL 打头,封死判决,无"但是"、无铺垫。先让他吞下死亡。
2. **尸检** — 单 idea 临床死因"死在动脉 X";有复发动脉则追加"和你上个 X idea 同一条动脉"(**可自验的 pattern,不是分数/进度/"你走得更远"** —— 跨次比较/记分会<1 秒被识破成 dark pattern,并让 KILL 显得可谈判)。**全栈成败押在这条尸检的精度上。**
3. **Pattern + 内核(条件触发)** — 可选:一条零解读的跨 idea 规律当裸数据,用户自己解读;真有强组件才存,**没有就明确"这里没什么可存的"**(这个**非触发**正是证明不是模板的东西)。
4. **Quest** — 收在可证伪阈值:"KILL,**除非**[idea 专属、推导出、预承诺、比裸 KILL 更难的 X];本周末零开发最便宜验证。" **X 钉死、不可谈判**,否则会变成偷偷收回裁决的逃生舱。它占住退出情绪 —— 用户带走一把更利的刀。

**已证伪的反例(别用)**: "快速证伪 IS 技能、你正在 get 它"(关于人、被 KILL 计数证伪)、"你的 idea 死得越来越晚=你内化了 filter"(夹带不可证伪因果)、闯关/XP/记分外壳。

**Open problem(尚未解决)**: 同一用户跨 N 次 KILL,即使条件触发,**四拍剪影本身**到第 3-4 次会被 pattern-match → 解法只能是**结构变异**,这是下一仗。

## 触发与 clarify-gate

- **触发**: 用户 brainstorm 新方向 / 问"能不能赚钱、是平台还是 agency、引擎在哪、冷启动能否打通" / 比较方向 / 上线前压测模型 → 自动触发**快筛**。说"深入/严格/多方案/帮我 brainstorm 并验证" → 升**深评**(跑 Workflow,见下)。
- **Clarify-gate(关键)**: 快筛前**先核对三件载重信息,缺则必须先问、不许猜** —— ①**供给侧**(谁是供给方?今天是否已在为别的目的营业?);②**真实 WTP / 付费方**(谁掏钱、为什么掏、口头还是真金?);③**创始人资产**(资金/人脉/分发/供给独家关系 —— 决定平台 vs agency、决定点火,且世上无现成数据,只能问)。**缺①②会算错天花板,缺③会把"对别人可行"错判成"对你可行"。** 其余信息可用区间/费米假设补,但**必须显式标注假设**,不得伪装成事实。审问创始人资产时**逼证据**(如"说出 5 个 week-1 就会答应的供给/分发的人名"),不接受"我人脉很好"。
- **澄清只能问「用户独占的事实」,绝不能甩分析活(强制 —— A14 J 实测)**: 一个澄清问题合法,当且仅当它问的是**只有用户知道、且能掀翻裁决的客观事实**(如在岸/离岸、有无内部供给渠道、真实成交价)。**禁止**把这三类伪装成澄清问题丢回去:①**判断/分析**("这个机制适不适用于你""你算得过来吗"——这是工具的活,自己算);②**让用户帮我证伪我自己的结论**("说一个能掀翻结论的 fact"——证伪是工具的活);③**可自查的 macro/合规**(自己查)。**铁律:凡载重的不确定,工具先把每种取值的分支都算好,把用户的回答压缩成「10 秒确认一个事实」,而不是「替工具做功课」。** 问错的代价是真实的:用户会正确地反问"那我干嘛不自己开 GPT 问"——一旦澄清像交作业,工具就没有比裸 LLM 多出任何价值。

## 深评模式怎么跑

用户要"深入/多方案/严格验证"时,用 Workflow 工具跑同目录的 `feasibility-workflow.js`(按需 Edit 注入本次的硬约束/市场/创始人资产),它会: brainstorm ≥5 方案 → 逐方案 8 维对抗打分 + 母级死因逃逸裁决 → loop-until-dry(连续两轮无 PROMISING 即停)→ 首席综合。跑完由你整合两轮、判断角度是否穷尽、产出完整报告。
